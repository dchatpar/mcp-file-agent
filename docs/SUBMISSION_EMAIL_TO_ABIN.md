**1. Assignment objective (what was built)**

I implemented a **LangChain code-first CLI agent** that satisfies the scenario in your Instructions.pdf:

- **In-process Local File Search MCP** (FastMCP) for sandboxed metadata search, PDF listing, PDF full-text search, and single-PDF read.
- **Remote Microsoft Learn MCP** at https://learn.microsoft.com/api/mcp using **streamable_http** (native Learn tools such as microsoft_docs_search).
- **SKILL-based routing** (SKILL.md in repo) with strict response contracts:
  - **Local file queries** ? JSON only (no markdown/prose).
  - **Microsoft / Azure documentation queries** ? Learn MCP only, answer capped at **2000 characters**.
  - **Out-of-scope queries** ? fixed JSON error (no tools):
    {"error": "Query out of scope. Only file search and Microsoft/Azure queries are supported."}

The interface is an **async REPL CLI** (command: file-search-agent), which meets the CLI acceptable requirement (no web UI).

---

**2. Architecture (high level)**

Flow: User query ? LangChain create_agent selects tools per SKILL.md ? tool execution (local in-process MCP or remote Learn) ? output_guard.py enforces JSON-only local output, prefers last local tool JSON when the model adds prose, truncates Learn answers, strips reasoning artifacts ? response printed to terminal.

Key modules:

| Component | Path | Role |
|-----------|------|------|
| CLI entry | src/file_search_agent/main.py | REPL loop, guard application |
| Agent factory | src/file_search_agent/agent_factory.py | create_agent, MCP clients, ChatOpenAI |
| Local MCP | src/file_search_agent/mcp/local_file_search.py | FastMCP tools + sandbox |
| Models | src/file_search_agent/models.py | Pydantic schemas + assignment JSON aliases |
| Routing / guard | routing.py, output_guard.py | Out-of-scope constant, JSON enforcement |
| Config | config.py | SEARCH_ROOT, API URL/model, Learn URL, 2000-char limit |

---

**3. Local MCP tools (assignment + enhancements)**

| Tool | Capability |
|------|------------|
| search_files | Filter by file name, folder path, extension, created/modified dates, min/max size |
| search_pdf_content | Full-text keyword search inside PDFs (pypdf) |

| read_pdf_content | Read text from a single PDF by relative path |

All local paths are resolved under SEARCH_ROOT (default: data/samples/zoology/). Path traversal outside the sandbox is rejected (covered by security tests).

**5. LLM configuration**

The agent uses LangChain ChatOpenAI against an OpenAI-compatible HTTP API.

- **Default (used for E2E verification):** MiniMax M2.7
  - OPENAI_BASE_URL=https://api.minimax.io/v1
  - OPENAI_MODEL=MiniMax-M2.7
  - MiniMax-specific extra_body (thinking disabled) is applied only when the base URL indicates MiniMax (OpenAI can be swapped without code changes).

- **OpenAI / GPT-5.x (assignment-aligned option):** Documented in repo ť copy .env.openai.example to .env and set OpenAI key, https://api.openai.com/v1, and model id (e.g. gpt-4o or GPT-5.x when available on the account).

1. Assignment objective

I implemented a LangChain code-first CLI agent aligned with the scenario in your Instructions.pdf:

  • In-process Local File Search MCP (FastMCP) ¯ sandboxed metadata search, PDF listing, PDF full-text search, and single-PDF read.
  • Remote Microsoft Learn MCP at https://learn.microsoft.com/api/mcp using streamable_http (native tools e.g. microsoft_docs_search).
  • SKILL-based routing (SKILL.md) with strict response contracts:
    • Local file queries ? JSON only (no markdown or prose).
    • Microsoft / Azure documentation ? Learn MCP only; answer capped at 2,000 characters.
    • Out-of-scope queries ? fixed JSON error; no tools invoked.
  • Interface: async REPL CLI (file-search-agent) ¯ meets the CLI-acceptable requirement (no web UI).

{"error": "Query out of scope. Only file search and Microsoft/Azure queries are supported."}

2. Architecture

Request flow: User query in REPL ? LangChain create_agent selects tools per SKILL.md ? local in-process MCP or remote Learn MCP executes ? output_guard.py enforces JSON-only local responses, prefers last tool JSON when the model adds prose, truncates Microsoft answers, strips reasoning artifacts ? guarded output printed to terminal.

Component Path Role
CLI entry src/file_search_agent/main.py REPL loop, output guard
Agent factory agent_factory.py create_agent, MCP clients, ChatOpenAI
Local MCP mcp/local_file_search.py FastMCP tools, sandbox
Models models.py Pydantic + assignment JSON aliases
Routing / guard routing.py, output_guard.py Out-of-scope constant, JSON enforcement
Config config.py SEARCH_ROOT, API, Learn URL, limits

3. Local MCP tools

5. LLM configuration

LangChain ChatOpenAI against an OpenAI-compatible HTTP API.

Default (E2E verified): MiniMax M2.7

  • OPENAI_BASE_URL=https://api.minimax.io/v1
  • OPENAI_MODEL=MiniMax-M2.7
  • MiniMax-only extra_body (thinking disabled) when base URL indicates MiniMax ¯ OpenAI swap documented without code changes.

OpenAI / GPT-5.x option: Copy .env.openai.example ? .env; set OpenAI key and model id. See docs/LLM_PROVIDER_GUIDE.md.

Security: API keys only in .env (gitignored; never committed).
